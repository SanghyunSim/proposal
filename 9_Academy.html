1. 목표 시스템 정의

1-Liner Definition

차세대 무역아카데미는 온·오프라인·Live·위탁교육·자격시험을 단일 LMS/LXP 플랫폼에서 통합 운영하고, 학습 이력·콘텐츠·결제·정산·자격증·상담 데이터를 KITA 내부 시스템 및 Trade Digital Brain과 연결하여 AI 튜터·맞춤형 과정 추천·교육 운영 분석까지 제공하는 데이터 기반 무역교육 플랫폼이다.

1.1 사업 배경 (Why)

  • 교육 운영 채널 분산: 무역아카데미는 무역교육, 인력매칭, 맞춤형 위탁연수, 전문인력 양성 기능을 수행하지만 온·오프라인 교육, 자격시험, 위탁교육, 결제/정산, 수료증/자격증 발급이 통합 경험으로 연결되지 않음 [RFP II-1, p.9]
  • 운영 자동화 부족: 과정 개설, 차수/기수 운영, 출결, 평가, 설문, 수료, 정산, 강사/멘토 관리가 업무별로 단절되어 교육담당자의 반복 운영 부담이 큼 [RFP III-자-SFR-AC-002~008, p.96~99]
  • AI 학습 지원 요구 확대: RFP는 학습자 프로필·진단 결과·학습 이력 기반 과정 추천, AI 튜터, 실시간 강의 요약, 동영상 자동 번역, Transcript 기반 구간 검색을 명시적으로 요구 [RFP III-자-SFR-AC-009, p.99~100]
  • 데이터 분석 기반 운영 전환 필요: 교육 성과, 설문, 매출/비용, 학습 행동 패턴을 분석하고 자연어 질의 기반 AI 통계 리포트를 생성해야 함 [RFP III-자-SFR-AC-006, p.98]
  • 내부 시스템 연계 복잡성: KNEX SSO, 회원회비시스템 할인, 멤버십 수료증/자격증, KAMIS 정산, 통합데이터플랫폼 연계를 동시에 구현해야 하며 연동 장애 로그와 모니터링이 필요 [RFP III-자-INR-AC-002, p.101]

1.2 구축 목적 (What)

RFP I-4 추진범위 아홉째 항목 [RFP I-4, p.4]:

"무역아카데미의 온·오프라인 등 다양한 교육 유형을 통합 운영할 수 있는 차세대 학습 플랫폼을 구축하고, 데이터 기반 교육 운영 체계와 사용자 중심 학습 환경 구현을 목적으로 함"

RFP I-4 과제 내용 [RFP I-4, p.4]:

"온·오프라인 교육과정 운영, 학습 콘텐츠 관리, 학습자 진도·평가 관리 기능을 포함한 사용자 대상 LMS 기반 학습 서비스 구축"

RFP I-4 과제 내용 [RFP I-4, p.4]:

"학습자 특성 및 학습 이력 기반 교육 과정 추천, AI 튜터 기반 학습 지원, 교육 운영 데이터 분석 및 리포트 기능을 포함한 AI 기반 교육 서비스 구축"

1.3 최종 지향점 (Goal)

  • Unified LMS/LXP Platform: 온라인·오프라인·Live·혼합형·패키지·위탁교육·자격시험을 단일 운영 모델로 관리하고, 수강신청-학습-평가-수료-증명-정산까지 끊김 없는 E2E 프로세스 제공 [RFP III-자-SFR-AC-002~005, p.96~98]
  • AI Learning Companion: AI 튜터가 교안·동영상·Transcript를 학습하여 Q&A, 강의 요약, 핵심정리, 자동 번역, 구간 검색, 맞춤 로드맵을 지원 [RFP III-자-SFR-AC-009, p.99~100]
  • B2B 위탁교육 운영 허브: 기업 담당자별 권한, 기업 전용 과정/요금, 일괄 수강생 등록, 청구·정산, 기업 성과 리포트를 제공하여 B2B 교육 매출과 운영 효율을 동시에 확대 [RFP III-자-SFR-AC-008, p.99]
  • Education Data Foundation: 학습 이력·평가·설문·상담·결제·자격 데이터를 표준화하여 Trade Digital Brain으로 송수신하고 교육 추천·대시보드·Agent의 공통 데이터 자산으로 활용 [RFP III-자-DAR-AC-001~002, p.102]
  • Mobile-First Learning Experience: PC·태블릿·스마트폰에서 동일 학습 기능을 제공하고 웹 표준·접근성·SEO·브랜드 커스터마이징을 반영 [RFP III-자-INR-AC-003~005, p.101~102]

3. 핵심 요구사항 및 필수 고려사항 TOP 5

차세대 무역아카데미는 단순 LMS 교체가 아니라 교육 운영 플랫폼 + B2B 위탁교육 + AI 학습 지원 + 결제/정산 + 데이터 분석이 결합된 복합 시스템이다. 평가 가중치는 기능 요구사항 9개 중 SFR-AC-002/003/006/008/009와 인터페이스·데이터 요구사항에 집중될 가능성이 높다.

KEY-1. 온·오프라인·Live·혼합형 교육과정 전주기 운영

근거: [RFP III-자-SFR-AC-002, p.96~97]

  • 과정 분류 체계를 계층형 Tree로 관리하고 노출 순서·활성/비활성 설정 필요
  • 과정명, 교육목표, 교육대상, 기간, 교육비, 맛보기 영상, 주차/차수/기수, 커리큘럼, 온라인/오프라인/Live 유형을 통합 관리
  • 과정 패키지, 선발 전형, 선수과목, 수료 기준, 콘텐츠 이어보기/드래그 제한 등 정책 설정 요구
  • QR 출결, Live 강의 접근 제어, 투표/반응, 시험/과제/토론, 설문, 게시판/커뮤니티, 운영 보고서까지 통합

난이도/배점 추정: ★★★★★

  • 요구 범위가 LMS Core, LXP, Live Class, Assessment, Community, Operation Admin을 모두 포함
  • 과정 모델링을 잘못하면 이후 위탁교육, 통계, 수료증, 결제, 추천이 모두 흔들림

구현 핵심 포인트:

  • Course Domain Model: Course → Curriculum → Module → Session → Activity → Assessment → Completion Rule
  • 과정 템플릿/복제: 반복 개설 업무를 줄이기 위해 과정 마스터와 기수 인스턴스를 분리
  • 출결 정책: QR, Live 입장 로그, VOD 시청 로그를 통합한 Attendance Event Model
  • 평가/수료 엔진: Rule DSL 기반으로 과정별 수료 기준을 코드 수정 없이 설정
  • 운영 자동화: 진도율·과제 미제출·설문 미참여 조건 기반 대상자 그룹핑 + 알림 자동 발송

KEY-2. 콘텐츠 관리 + STT/Transcript + AI 튜터 기반 학습 지원

근거: [RFP III-자-SFR-AC-003, p.97] + [SFR-AC-009, p.99~100]

  • PDF, MS Office, 이미지, 동영상, 외부 링크, SCORM 등 다양한 포맷 등록·관리
  • 동영상 업로드 후 자동 인코딩, STT 변환, 자막/목차 생성, 검색 키워드 메타데이터 활용
  • AI가 Transcript를 분석하여 검색 키워드 등장 구간을 식별하고 해당 지점으로 이동 지원
  • AI 튜터가 교육 콘텐츠를 학습하여 질문에 응답하고, Live/온라인 강의 요약·핵심정리·자동 번역 제공

난이도/배점 추정: ★★★★★

  • AI 튜터 품질은 교육 플랫폼 차별화의 핵심. 단순 챗봇으로 구현하면 평가에서 약함
  • 영상 STT·자막·구간 검색·RAG·번역·저작권/접근권한을 동시에 다뤄야 함

구현 핵심 포인트:

  • Media Pipeline: 업로드 → 바이러스 검사 → Transcoding(HLS) → STT → Chunking → Embedding → Search Index
  • STT: Whisper large-v3 / Naver CLOVA Speech / Google STT 중 한국어 정확도와 보안 기준으로 선택
  • AI Tutor: 과정/차시 권한을 검증한 뒤 해당 콘텐츠 범위에서만 RAG 응답
  • Transcript Search: OpenSearch + Vector DB 하이브리드 검색, 타임코드 기반 영상 점프
  • 자동 번역: 원문/번역문 버전 관리 + 사람 검수 상태값 제공
  • 저작권/보안: Signed URL, DRM/Watermark, 다운로드 권한 분리, 외부 LLM 학습 차단

KEY-3. 위탁교육 B2B 운영 + 기업별 권한·요금·성과 리포트

근거: [RFP III-자-SFR-AC-008, p.99]

  • 기업별 담당자 계정을 여러 명 발급하고 해당 기업 소속 직원 데이터만 조회
  • 기업 전용 과정 매핑, 타 기업/일반 회원 노출 차단, 기업별 할인율/수강료 정책 설정
  • 기업 단위 수강생 일괄 등록(엑셀 업로드), 기업용 공지/교육자료 별도 제공
  • 월별/기수별 위탁 교육비 자동 산출, 계산서 발행용 데이터 생성, 기업/부서/직무별 성과 비교

난이도/배점 추정: ★★★★☆

  • B2C LMS와 B2B 교육 운영의 권한/정산/리포팅 모델이 충돌하기 쉬움
  • 기업별 데이터 격리와 운영자/기업담당자/수강생 권한 범위가 평가 시 주요 질문이 될 가능성 높음

구현 핵심 포인트:

  • Tenant-like 기업 스코프: company_id 기반 Row-Level Security와 API Scope 적용
  • 기업 전용 카탈로그: 과정 노출, 요금, 쿠폰, 신청서 템플릿을 기업별 Override
  • Bulk Enrollment: 엑셀 업로드 검증, 중복/오류 Preview, Dry-run 후 확정 반영
  • B2B Billing: 월별/기수별 청구 산식 버전 관리 + KAMIS 전표 연계
  • Enterprise Report: 수강률, 수료율, 만족도, 참여도, 부서/직무별 성과 비교

KEY-4. 결제·정산·회원사 할인·자격증 발급 연계

근거: [RFP III-자-SFR-AC-004~005, p.97~98] + [INR-AC-002, p.101]

  • 시험 접수/결제/환불, 성적 공개, 이의신청, 합격자 관리, 자격증 발급번호 부여 및 재발급 이력 관리
  • 교육 과정/패키지 구매, 환불, 세금계산서/영수증 자동 발급, 쿠폰/할인, 제휴 마케팅 정산
  • 회원회비시스템으로 회원사별 회비 납부 현황을 확인하여 수강료 할인 대상 여부를 실시간 적용
  • KAMIS와 예산코드, FB 입금, 직계좌 입금, 전표 관리 등 정산 데이터 연동
  • 멤버십서비스와 수료 이력 및 자격 취득 데이터를 연계하여 수료증·자격증 발급

난이도/배점 추정: ★★★★☆

  • 금전·자격·정산 데이터는 오류 허용도가 낮고 감사 추적 요구가 큼
  • 결제 실패, 환불, 할인 재계산, 전표 불일치, 자격증 재발급 같은 예외 플로우가 많음

구현 핵심 포인트:

  • Order State Machine: 신청, 결제대기, 결제완료, 환불요청, 부분환불, 취소, 정산완료 상태 표준화
  • Discount Engine: 회원사 여부, 재직증명, 쿠폰, 패키지, 기업별 요금 정책 우선순위 정의
  • Payment Outbox Pattern: 결제 성공 후 수강권 부여/영수증/전표/알림을 이벤트 기반으로 보장
  • Audit Ledger: 결제·환불·할인·자격증 발급·성적 정정 이력 Append-only 저장
  • Reconciliation: PG, KAMIS, 내부 주문 DB 간 일/월 단위 대사 리포트 자동 생성

KEY-5. 데이터 표준화·이관·AI 분석 리포트·데이터 주권

근거: [RFP III-자-SFR-AC-006, p.98] + [DAR-AC-001~003, p.102]

  • 회원, 과정, 수료, 평가, 설문, 매출, 비용, 상담, 콘텐츠 이용 데이터를 표준화해야 함
  • 기존 시스템 핵심 데이터를 분석하여 신규 시스템으로 이관하고 샘플 검증을 통해 정확성 확인
  • 통합 데이터 플랫폼(Data Lake) 및 AI 분석 환경에서 데이터를 활용할 수 있도록 표준 기반 인터페이스 제공
  • 교육담당자가 자연어로 데이터 조회를 요청하면 시스템이 의도를 파악해 수치와 차트를 생성하고 엑셀/PDF 다운로드 제공
  • 무역아카데미 운영 중 생성된 모든 데이터 소유권은 무역협회에 있으며, 계약 종료 시 범용 포맷으로 반환 필요

난이도/배점 추정: ★★★★★

  • 학습 데이터는 AI 추천·대시보드·Agent 품질의 원천. 이관 품질이 낮으면 전체 서비스 신뢰도가 하락
  • 자연어 통계 리포트는 NL2SQL 안전성, 권한, 지표 정의, 차트 선택 로직이 함께 필요

구현 핵심 포인트:

  • Education Canonical Model: 회원, 과정, 콘텐츠, 학습활동, 평가, 설문, 결제, 자격증 표준 엔티티 정의
  • Migration Factory: 프로파일링 → 매핑 → 정제 → Dry-run → 샘플 검증 → 전체 이관 → Reconciliation
  • BI Semantic Layer: KPI(수료율, 만족도, 이탈률, 매출, 수익성)를 Metric Store로 정의
  • NL2SQL Guardrail: 허용된 Semantic Model 안에서만 질의, 민감정보 마스킹, 쿼리 비용 제한
  • Data Ownership: CSV/Parquet export, 메타데이터 사전, 삭제/반환 증적 자동 산출

5. 차별화된 제안 전략 (Value Proposition)

V-1. "LMS 교체"가 아니라 "무역교육 LXP + 운영 ERP"로 제안

  • 일반 LMS 패키지 도입 관점에서 벗어나 학습자 경험(LXP), 교육 운영 ERP, B2B 위탁교육, 자격시험, 결제/정산, AI 튜터를 하나의 도메인 아키텍처로 제시
  • 과정/차수/기수/콘텐츠/평가/수료/정산 엔티티를 분리해 향후 신규 교육상품을 빠르게 출시할 수 있는 구조 강조
  • 임원 설득 메시지: "운영 인력 증원 없이 과정 수와 수강생 수를 늘릴 수 있는 플랫폼"

V-2. AI 튜터를 "강의 콘텐츠 근거 기반"으로 제한하는 Trustworthy Learning AI

  • AI 튜터가 임의 지식을 답하는 구조가 아니라, 수강 권한이 있는 교안·영상 Transcript·FAQ·과정 공지 범위에서만 답변
  • 답변마다 출처 콘텐츠, 차시, 영상 타임코드, 문서 페이지를 제공하여 신뢰성 확보
  • Live 강의 종료 후 자동 요약·핵심정리·복습 퀴즈·학습 노트 생성으로 학습 몰입도 강화

V-3. 교육 데이터 Flywheel 구축

  • 학습 이벤트(검색, 시청, 북마크, 질문, 이탈, 반복 시청)를 표준 이벤트로 수집
  • 수집 데이터는 과정 추천, 이탈 위험 알림, 콘텐츠 개선, 강사 평가, 기업별 성과 리포트로 재활용
  • 통합대시보드 W5-2 교육·과정추천, W5-3 나의 학습·수료현황과 직접 연결하여 KITA 전체 플랫폼 개인화 강화 [ISP p.64~66]

V-4. 위탁교육 B2B 매출 확대를 위한 Enterprise Academy

  • 기업별 전용 카탈로그, 담당자 포털, 대량 등록, 기업별 요금, 월별 청구, 성과 리포트를 제공
  • 기업 담당자가 "우리 회사 직원의 교육 현황과 역량 향상"을 직접 확인하게 만들어 재계약/반복 구매 가능성 확대
  • 기업별 RLS와 청구 산식 버전 관리로 감사 가능한 B2B 운영 체계 구현

V-5. Safe Migration + Parallel Run 전략

  • 기존 아카데미 데이터의 과정/수강/결제/성적/자격/콘텐츠를 표준 모델로 매핑하고, 핵심 회차는 병행 검증
  • 수강신청-결제-학습-수료 업무 흐름을 실제 운영 시나리오로 리허설하여 전환 리스크 최소화
  • 데이터 반환/주권 요구를 충족하기 위해 이관 산출물과 Export 포맷을 프로젝트 초기부터 정의

7. 결론 및 제안 핵심 메시지

7.1 사업 본질 재정의

차세대 무역아카데미는 교육 콘텐츠를 온라인으로 제공하는 단순 LMS가 아니라, 무역 인재 양성 데이터를 KITA 전사 플랫폼의 핵심 자산으로 전환하는 교육 데이터 허브다.

7.2 제안 5축 메시지

  1. 운영 모델 표준화: 과정·기수·콘텐츠·평가·수료·정산을 표준 도메인 모델로 재설계
  2. AI 학습 경험: STT/Transcript/RAG 기반 AI 튜터로 학습 질문, 요약, 번역, 구간 검색 지원
  3. B2B 위탁교육 강화: 기업별 권한·요금·청구·성과 리포트로 반복 매출 기반 확대
  4. 정산/자격 신뢰성: 결제·환불·KAMIS·멤버십 자격증 발급을 감사 가능한 이벤트/원장 구조로 구현
  5. 전사 데이터 연계: 학습 이력과 과정 추천을 Trade Digital Brain, Trade Agent, 통합대시보드와 연결

7.3 평가표 매핑

7.4 Trade Digital Brain · Trade Agent · Dashboard와의 시너지

  • Trade Digital Brain: 학습 이력, 과정 메타, 설문, 콘텐츠 이용 로그를 통합 데이터 플랫폼에 적재하여 회원사 역량 분석과 교육 추천의 기반으로 활용
  • Trade Agent: 무역아카데미 AI 튜터, 교육상담 Agent, 교육관리자 Agent가 과정/콘텐츠 지식을 RAG로 활용
  • Trade Dashboard: W5-2 교육·과정추천, W5-3 나의 학습·수료현황 위젯으로 학습 진행과 추천 과정을 전사 개인화 화면에 노출
  • KITA.net/KNEX: SSO와 회원사 정보, 회비/등급 기반 할인 정책을 적용하여 학습 경험과 회원 혜택을 연결
수주 전략 한 줄 메시지: "우리는 무역아카데미를 LMS로 교체하는 것이 아니라, KITA 회원사의 수출 역량을 데이터로 진단하고 AI로 끌어올리는 차세대 무역교육 운영 플랫폼으로 구축한다."
