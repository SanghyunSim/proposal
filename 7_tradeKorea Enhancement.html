1. 목표 시스템 정의

1-Liner Definition

tradeKorea 기능 고도화는 KITA의 B2B 글로벌 매칭 플랫폼인 tradeKorea를 AI 기반 상품 등록·Inquiry 스코어링·Live Chat·Trade Agent 매칭으로 재구축하고, 모바일 앱을 오프라인 행사·생체인증·개인화 Push 중심으로 재구축하며, 견적-계약-e-Invoicing 모듈을 신규 추가해 매칭 이후 거래 전주기까지 확장하는 KITA 글로벌 무역 사업의 핵심 운영 플랫폼 고도화 사업이다.

1.1 사업 배경 (Why)

  • B2B 매칭 경쟁력 정체: tradeKorea는 KITA의 대표 B2B 매칭 플랫폼이지만 알리바바·EC21 등 글로벌 경쟁사 대비 AI 기반 매칭·진성 회원 검증·다국어 대응 측면에서 뒤처짐 [RFP I-4-사, p.4]
  • 허수 회원·INQ 신뢰도 문제: 바이어/셀러 DB의 허수 회원·저품질 Inquiry로 인해 매칭 결과 신뢰도 저하. 진성 검증 체계 부재 [RFP III-사-CSR-TK-002, p.86]
  • 모바일 사용성 미흡: 현재 tradeKorea 모바일은 단순 Web 모바일 변환 수준. 알림·개인화·오프라인 행사 연계가 없음 [RFP III-사-SFR-TK-002, p.87]
  • 매칭 이후 거래 단절: 바이어/셀러 매칭 후 견적·계약·인보이스 단계는 외부 도구로 이동 → 거래 전주기 데이터가 KITA에 축적되지 않음 [RFP III-사-SFR-TK-003, p.88]
  • 해외 사용자 접속 지연: 글로벌 바이어가 사용하는 플랫폼이지만 해외 지역 응답시간 SLA가 정의되어 있지 않음. 동남아·미주·유럽 사용자의 첫 세션 연결 지연 빈번 [RFP III-사-PER-TK-001, p.88]
  • 상품 데이터 품질 분산: 셀러가 등록하는 상품 페이지의 항목별 입력 수준 편차가 커서 검색·매칭 정확도 저하. 데이터 품질 거버넌스 체계 부재 [RFP III-사-CSR-TK-001, p.86]

1.2 구축 목적 (What)

RFP I-4 추진범위 일곱째 항목 [RFP I-4, p.4]:

"B2B 마케팅 플랫폼 경쟁력 제고를 위해 AI 기반으로 플랫폼 바이어/셀러 매칭 기능 강화, 모바일 앱 재구축, 데이터 기반 플랫폼 운영 체계를 구축하고 향후 글로벌 무역 확장을 위한 모델 설계. 견적·계약·물류·해외통관을 연계한 글로벌 무역 확장 모델 설계"

CSR-TK-001 명문 [RFP III-사-CSR-TK-001, p.86]:

"회원사 관리 정책 수립 — 바이어, 셀러별 관리 필요 마스터 데이터 정의 + 회원 라이프사이클(가입–활성–휴면–탈퇴) 단계 정의 + 활성화 회원 등급 재정립 + 비활성화 회원 구분 + 정기적 회원 데이터 점검 및 클렌징 기준"

1.3 최종 지향점 (Goal)

  • AI-First B2B Platform: 상품 등록(이미지·카탈로그→스펙·키워드 자동), Inquiry 작성·매칭·진정성 스코어링, Live Chat 초기 응답이 모두 AI 기반으로 동작 [RFP III-사-SFR-TK-001, p.86~87]
  • End-to-End Trade Lifecycle: 매칭(매칭) → 견적(Quote) → 계약(Contract) → 인보이스(e-Invoice)의 거래 전주기를 단일 플랫폼에서 처리 [RFP III-사-SFR-TK-003, p.88]
  • Mobile-First Field Engagement: 오프라인 무역 행사·전시회 현장에서 바이어와 셀러가 즉시 만나고 사후 Inquiry Chat으로 연결되는 현장 중심 모바일 UX [RFP III-사-SFR-TK-002, p.87]
  • Global Performance SLA: 해외 지역(동남아·미주·유럽·중국) 응답시간 정량 SLA + Multi-Region Edge 배포 + 진성 회원 글로벌 신뢰도 [RFP III-사-PER-TK-001, p.88]
  • Trade Agent Integration: AI 매칭·키워드·상품 특화 정보 자동 도출은 Trade Agent SFR-AP-006(추천형 Agent) + SFR-AP-009(서비스특화 Agent: 바이어/셀러 추천)로 직결 [RFP III-다-SFR-AP-006/009, p.54~58]

3. 핵심 요구사항 및 필수 고려사항 TOP 5

본 영역은 7개 요구사항으로 규모는 작지만 AI 매칭 정확도·거래 전주기 확장·해외 성능·진성 회원 거버넌스·모바일 현장 UX가 평가 핵심 가산점.

KEY-1. AI 기반 상품 등록·Inquiry Scoring·Live Chat (SFR-TK-001)

근거: [RFP III-사-SFR-TK-001, p.86~87]

  • AI 기반 상품 사진·카탈로그 → 주요 스펙·상세 설명 등 필드 입력 자동화
  • 검색에 활용될 상품 키워드 자동 도출
  • 상품 등록 단계별 미리 보기 지원
  • 실시간 메시징 Live Chat 일원화 + Direct 채팅 + 양식 고도화
  • AI 기반 초기 대응 (최초 문의 작성 시)
  • INQ 단위 채팅 히스토리 관리
  • Trade Agent 연계: AI 기반 Inquiry 스코어링·진정성 판단, 키워드·상품 특화 정보 자동 도출, 검색·INQ·매칭 이력 기반 정보 연계

난이도/배점 추정: ★★★★★

  • tradeKorea 영역의 핵심. AI 기능이 본 영역 평가의 50% 이상
  • 상품 등록 자동화 정확도(Vision Multimodal) + Inquiry 진정성 스코어링(허수 차단)이 본 영역 운영 KPI 결정
  • 평가표 "기술 및 기능 25점"의 직접 가산

구현 핵심 포인트:

  • 상품 등록 자동화: Vision Multimodal LLM (GPT-4o Vision / Claude 3.5 Sonnet / EXAONE Vision) + LayoutLM(카탈로그 PDF) → 스펙·키워드·카테고리 추출
  • 키워드 도출: BGE-M3 임베딩 + 검색 로그 분석 → SEO·매칭용 키워드 자동 제안
  • Inquiry Scoring: LightGBM 기반 분류 (특성: 회원 등급·KYB·과거 INQ 패턴·텍스트 품질·발신 패턴) → 진정성 점수(0~100)
  • Live Chat: WebSocket + Spring Cloud Stream + AI 초기 응답(Trade Agent SFR-AP-004 대화형 Agent)
  • 다국어: 자동 번역(BGE-M3 multilingual + Trade Agent 다국어 Agent SFR-AP-009)
  • 채팅 히스토리: PostgreSQL + Object Storage (첨부 파일) + 검색용 OpenSearch

KEY-2. Quote-Contract-e-Invoice 거래 전주기 확장 (SFR-TK-003)

근거: [RFP III-사-SFR-TK-003, p.88]

  • 견적: 표준 템플릿·Cost Breakdown·견적 시뮬레이션·버전 관리 + 상태 조회·이력(전송/수정/합의/만료)
  • 계약: 합의된 견적 기반 자동 생성 + 표준 계약 템플릿 + 독소조항 검토 + 전자서명
  • e-Invoice: 계약/주문 정보 기반 발행 + 국가별 규제 준수 + 발행·전달·보관·조회 일원화
  • 기존 모듈 연계 + 서비스 성과 KPI

난이도/배점 추정: ★★★★★

  • KITA가 단순 매칭 플랫폼에서 거래 플랫폼으로 도약하는 결정적 영역
  • e-Invoicing의 국가별 규제(PEPPOL, 미국 X12, EU EN16931, 베트남 Resolution, 사우디 ZATCA 등)는 글로벌 규제 대응의 정점
  • 전자서명·독소조항 검토는 법률 전문성 + 도메인 깊이 동시 요구

구현 핵심 포인트:

  • 견적 모듈: 표준 템플릿(JSON Schema) + Cost Breakdown 트리 구조 + 시뮬레이션(통화·관세·물류 변수) + 버전 관리(Git-like Diff)
  • 계약 모듈: 계약 자동 생성 (견적 → 변수 치환 → 표준 템플릿) + 독소조항 검토 (LLM 기반 위험 조항 탐지 + Trade Agent 법률 Agent 연계)
  • 전자서명: DocuSign API(글로벌 표준) + 한국 회원사용 KICA 공인 PKI + 글로벌 호환 ESIGN/eIDAS
  • e-Invoicing: PEPPOL Network (AS4 프로토콜, OpenPEPPOL 표준) + 국가별 어댑터(EU EN16931, 미국 X12, 베트남, 사우디 등)
  • 인보이스 보관: 5년+ 무결성 보관 (Object Storage WORM)
  • 성과 KPI: 견적 → 계약 전환율, 계약 → 인보이스 전환율, 평균 거래 사이클 타임

KEY-3. 해외 사용자 성능 + 글로벌 Edge (PER-TK-001)

근거: [RFP III-사-PER-TK-001, p.88]

  • 현행 Web 서비스 환경(설정·리소스 경로·도메인) 최적화
  • 해외 사용자 접속 지연(최초 세션 연결 등) 현상 해소
  • 해외 지역별 목표 응답시간 기준 수립

난이도/배점 추정: ★★★★☆

  • tradeKorea는 글로벌 바이어/셀러가 사용하는 플랫폼 — 해외 응답 SLA가 사업의 본질적 SLA
  • 평가표 "성능 및 품질 15점"의 직접 매핑

구현 핵심 포인트:

  • 글로벌 CDN: Akamai(KITA 기존) 메인 + Cloudflare(보조) → 정적 자원·이미지 전 세계 배포
  • Multi-Region Edge: KT Cloud + AWS Global Accelerator 활용 → 동남아(SG)·미주(US-East)·유럽(EU-West)·중국(별도 ICP) 각 리전 Edge POP
  • TLS 최적화: TLS 1.3 + 0-RTT + OCSP Stapling
  • 자원 최적화: 이미지 WebP/AVIF + Lazy Loading + Code Splitting + Brotli 압축
  • DNS: GeoDNS (Route 53 / Akamai EdgeDNS) → 가까운 리전 자동 라우팅
  • 응답시간 SLA 정량 약속:
  • 한국 P95: 1.5초
  • 동남아·중국 P95: 2.5초
  • 미주·유럽 P95: 3초
  • 첫 세션 연결(TLS Handshake): P95 < 500ms
  • 모니터링: 글로벌 Synthetic Test (Catchpoint/Pingdom) + RUM(Real User Monitoring)

KEY-4. 회원·상품 데이터 거버넌스 + 진성 검증 (CSR-TK-001 + 002)

근거: [RFP III-사-CSR-TK-001, p.86] + [CSR-TK-002, p.86]

  • 회원 라이프사이클(가입-활성-휴면-탈퇴) + 등급 재정립 + 비활성 회원 구분(휴면·제재) + 정기 클렌징
  • 바이어/셀러 차별화 관리 기준 + 셀러 인증(Certificate)
  • 상품 데이터 품질 기준(필수/선택·최소 입력 요건) + 상태 정의(거래 가능/검수 중/장기 미활성/거래 불가) + 점진적 품질 상향
  • 진성 바이어/셀러 검증 체계 + 검증 정보 정의 + 관리 방안

난이도/배점 추정: ★★★★☆

  • 컨설팅 영역 — 평가표 "전략 및 방법론 15점"
  • 진성 검증은 글로벌 B2B 플랫폼의 핵심 차별화. 알리바바 등 경쟁사 대비 KITA의 신뢰도 무기
  • 회원 클렌징은 회원 이탈 리스크 동반 → 거버넌스 정책의 정교함이 평가 가산

구현 핵심 포인트:

  • 상품 품질 점수: 필수 항목 충족률 + 이미지 수·해상도 + 설명 풍부도 + 인증 마크 → 검색 노출 가중치

  • 상품 상태 머신: 거래 가능 ↔ 검수 중 ↔ 장기 미활성 ↔ 거래 불가 자동 전이

KEY-5. 모바일 + 오프라인 행사 현장 연계 (SFR-TK-002)

구현 핵심 포인트:

5. 차별화된 제안 전략 (Value Proposition)

V-1. AI-Native B2B Matching — "허수 INQ를 시스템이 먼저 거른다"

고객 Pain: 해외 바이어가 보내는 Inquiry의 상당수는 스팸·광고·허수. 셀러가 모든 INQ에 대응하면 시간 낭비, 무시하면 진성 기회 손실.

제안 해법:

  • AI Inquiry Scoring (LightGBM) → 진정성 점수 표시 + 자동 필터링 임계치
  • AI Live Chat 초기 응답 → 셀러 본격 대응 전 1차 검증
  • 키워드·상품 특화 정보 자동 도출 → 매칭 정확도 향상
  • 분기별 INQ 전환율 + 진성 비율을 발주처와 합동 리뷰
  • 차별화 메시지: "셀러는 진성 INQ에만 집중하고 허수는 AI가 먼저 거른다"

V-2. Match-to-Cash Lifecycle — "매칭 이후 거래 전주기를 KITA가 책임진다"

고객 Pain: tradeKorea가 매칭만 제공하면 매칭 이후 거래는 외부 도구로 이동 → KITA에 데이터·수수료·차별화가 남지 않음.

제안 해법:

  • 견적-계약-e-Invoice 모듈로 매칭 이후 거래 전주기를 단일 플랫폼화
  • 견적 자동 생성 → 계약 자동 채움 → 인보이스 자동 발행의 데이터 연속성
  • PEPPOL 글로벌 표준 + DocuSign으로 글로벌 호환성
  • 거래 전주기 데이터를 Trade Digital Brain에 적재 → 추천 학습·국가별 통계 분석
  • 차별화 메시지: "KITA에서 만난 바이어와의 거래가 KITA에서 끝난다"

V-3. Field-First Mobile — "오프라인 전시회를 디지털 자산으로 전환"

고객 Pain: KITA는 다수 오프라인 전시회·박람회를 운영하지만 현장 매칭 데이터가 시스템에 축적되지 않음.

제안 해법:

  • 모바일 앱 + BLE 비콘 + QR 체크인으로 현장 셀러·바이어 자동 매칭
  • 전시 부스 방문 → 자동 INQ Chat 생성 → 사후 추적
  • 만족도 조사 + Trade Agent 분석 → 행사 ROI 정량 측정
  • 차별화 메시지: "전시회가 끝나도 매칭은 계속된다"

V-4. Global SLA Performance — "해외 응답시간을 정량 약속한다"

고객 Pain: 글로벌 바이어가 사용하는 플랫폼인데 해외 응답시간 SLA가 정의되어 있지 않음.

제안 해법:

  • 지역별 P95 SLA 정량 명시(한국 1.5초/동남아 2.5초/미주·유럽 3초)
  • Akamai + Multi-Region Edge + GeoDNS로 글로벌 가속
  • 분기별 RUM(Real User Monitoring) 리포트 + Synthetic Test 결과 발주처 공유
  • 미달 시 자동 알림 + 우선 개선
  • 차별화 메시지: "글로벌 사용자에게 글로벌 SLA로 답한다"

V-5. Trust-Verified Membership — "진성 회원이 곧 KITA의 차별화"

고객 Pain: 알리바바·EC21 등 글로벌 경쟁사 대비 KITA의 차별화는 "공공기관의 신뢰" — 그러나 진성 검증 시스템이 없으면 신뢰가 추상화됨.

제안 해법:

  • KYB 자동 검증 + 셀러 인증 마크(Verified Seller) + 수출실적 자동 표시
  • 회원 등급 재정립 (Bronze/Silver/Gold/Platinum) + 등급별 검색 노출 가중치
  • 분기 정기 클렌징 + 휴면·제재 자동 처리
  • "신뢰 점수"를 회원 카드에 시각화
  • 차별화 메시지: "KITA tradeKorea의 셀러는 정부 인증 검증 회원이다"

7. 결론 및 제안 핵심 메시지

7.1 사업 본질 재정의

tradeKorea 기능 고도화는 "기존 시스템 고도화"가 아니라, "AI 매칭 + 거래 전주기(견적·계약·인보이스) + 글로벌 Edge 성능 + 진성 회원 거버넌스"가 한 시스템으로 통합된 KITA 글로벌 무역 사업의 핵심 운영 플랫폼 재구축이다. 단순 기능 개선이 아니라 KITA가 알리바바·EC21과 경쟁할 수 있는 글로벌 B2B 플랫폼으로의 도약.

7.2 제안 5축 메시지

  1. AI-Native B2B Matching — 허수 INQ를 시스템이 먼저 거른다
  2. Match-to-Cash Lifecycle — 매칭 이후 거래 전주기를 KITA가 책임진다
  3. Field-First Mobile — 오프라인 전시회를 디지털 자산으로 전환
  4. Global SLA Performance — 해외 응답시간을 정량 약속한다
  5. Trust-Verified Membership — 진성 회원이 KITA의 글로벌 차별화
