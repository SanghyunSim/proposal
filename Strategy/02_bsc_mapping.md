## 1. BSC 설계 전제

### 1.1 설계 배경

본 BSC는 **단일 BSC가 두 시간축(Bid/Delivery)을 동시에 다루는 이중 구조**로 설계한다. 이유는 다음과 같다.

- Bid 단계의 산출물(제안서 품질·SME 확보)이 곧 Delivery 단계의 실행 자산이 되므로 두 BSC를 분리하면 학습·성장 자산이 단절된다.
- 그러나 KPI 측정 주기·책임 조직·후행 시점이 다르므로 **각 KPI에 Phase 태그(B/D/B+D)**를 부여하여 운영상으로는 분리 가능하게 한다.
- 가격 전략 KPI는 OKR이 다루지 못한 영역이므로 BSC에서 명시 보강한다 (재무 관점 + 내부 프로세스 관점).
- 도메인 약점 극복 속도는 학습·성장의 핵심 선행 지표로 위치시킨다.

### 1.2 인과관계 가설 (전략 논리)

> **"도메인 SME·AI 역량을 빠르게 내재화(L&G) → 제안 품질·기술 차별화로 전환(IP) → KITA·사용자 만족 확보(C) → 수주·후속 매출 실현(F)"**

이 가설을 BSC 전 KPI가 직간접으로 뒷받침하도록 설계했다.

### 2.2 전략맵 해설

- **학습·성장이 출발점**: 도메인 약점이라는 구조적 한계는 SME 확보 속도·사내 인증 인력 증가·기술 자산 축적으로 1차 해소된다.
- **내부 프로세스가 변환기**: L&G 자산을 제안서 품질·시스템 SLA·가격 모델 정확도로 변환한다. 가격 KPI를 IP 관점에 두는 이유는 가격 산정 자체가 BoQ·원가 산출 프로세스의 결과물이기 때문이다.
- **고객 관점이 검증대**: 제안 평가위원(Bid)→KITA 실무·임원(Delivery)→무역업계 회원(End-User)의 3계층 고객을 모두 측정한다.
- **재무 관점이 최종 성과**: 단기 수주액·중기 수익률·장기 후속 파이프라인의 시차 다른 3개 지표로 측정한다.

- F4는 Lead 지표로 위치시켜 Delivery 후반의 영업 활동을 사전 견인.

### 5.1 핵심 가설 매트릭스

| 선행 KPI | → | 후행 KPI | 가설 근거 |
|---------|---|---------|----------|
| LG1 (SME 확보) | → | IP1 (요구사항 A+ 비율) | SME 자문이 187개 요구사항 응답의 도메인 정합성을 직접 향상시킨다 |
| LG3 (도메인 인사이트) | → | C2 (차별화 명확성) | 인사이트 30개가 제안서 "현업 페인포인트 분석"의 깊이를 결정한다 |
| LG2 (사내 인증 인력) | → | IP5 (Agent 정확도) | 무역 인증 인력이 Agent 검수와 도메인 룰 정의 품질을 좌우한다 |
| LG4 (재사용 자산) | → | IP4 (마일스톤 적기) | 자산화 50건이 다음 분기 일정 단축의 직접 동인이 된다 |
| LG5 (파트너십 생태계) | → | C4 (사용자 활성화율) | 회원사 자문단 20명의 채널이 44만 회원 채택 곡선을 견인한다 |
| IP1, IP2 (제안 품질) | → | C1, C2 (평가 점수) | 요구사항 A+ 75% 및 데모 8건이 평가위원 정성 점수의 핵심 인자 |
| IP3 (가격 정확도) | → | F3 (가격경쟁력 지수) | BoQ ±5% 정확도가 가격 안전대(0.92~0.97) 진입의 전제 |
| C1, C2 (평가 우위) | → | F1 (수주 계약액) | 기술 80% 가중치 구조에서 평가 점수가 수주 확률을 결정 |
| C3 (KITA 만족도) | → | F4 (후속 파이프라인) | 만족도 4.5+가 후속 사업 추천·재발주의 선행 조건 |
| C4, C5 (사용자 채택·NPS) | → | F2 (수익률) | 사용자 활성화가 변경요청(CR) 감소 → 마진 보호 |
| IP4, IP5, IP6 (SLA·결함) | → | C3 (만족도) | 약속한 일정·품질이 KITA의 신뢰의 직접 원천 |

### 5.2 가설 검증 방법

- 분기 BSC 리뷰에서 선행 KPI의 변화량 → 후행 KPI 변화량의 상관관계를 추세 비교.
- 상관관계가 약한 가설은 다음 분기에 다음 중 1개 조치: ①가설 수정 ②선행 KPI 재정의 ③측정 주기 단축.

## 7. SWOT 전문가 전달 사항

본 BSC 작성 과정에서 식별된 SWOT 분석 보강 요청 사항:

### 7.1 약점(W) 후보 (BSC 사각지대에서 발견)

- **W 후보 1: 직원 EX/이탈 리스크** — KR-D4.4가 OKR에 있으나 BSC에서는 별도 KPI로 잡지 않음. 이는 본 BSC가 외부(시장·고객) 시각의 균형에 우선순위를 둔 결과. SWOT의 W에서 "AX/AI 인력 시장 과열로 인한 핵심 인력 이탈 리스크"로 별도 식별 요청.
- **W 후보 2: 가격 산정 모델의 검증 이력 부족** — F3·IP3에서 가격 KPI를 신설했으나, 우리 회사가 대형 공공 AX 사업의 가격 산정 트랙레코드가 부족할 가능성. SWOT에서 W로 평가 요청.
- **W 후보 3: 도메인 인사이트 라이브러리 운영 경험 부족** — LG3을 outcome으로 잡았으나, 이 라이브러리를 지속 관리하는 KM(지식관리) 체계 자체가 약점일 수 있음.

### 7.2 위협(T) 보강 요청

- **T 보강 1: ISP 수행사의 "현행 답습 우위"가 평가위원에게 안정성으로 평가될 위험** — 본 BSC는 혁신/차별화에 가중치를 두지만, 평가위원 다수가 보수적이면 오히려 ISP 수행사의 점수가 높을 수 있음. SWOT의 ST 전략에서 다룰 것.
- **T 보강 2: 가격평가 20점에서 ISP 수행사가 원가 우위 가능성** — F3(가격경쟁력 지수)와 직결되는 외부 위협.

### 7.3 강점(S) 활용 강화 요청

- LG1·LG3에서 "도메인 약점 → 학습 자산화"의 속도가 곧 차별화임을 SWOT의 WO 전략(약점 활용 기회)에 명시 반영 요청.
