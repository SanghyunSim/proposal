1. 목표 시스템 정의

1-Liner Definition

수출실적증명 자동화서비스는 KITA 멤버십 회원사가 신청하는 용역·전자적무체물 수출실적증명서의 신청·증빙등록·심사·발급 전 과정을 OCR·문서이해 AI·도메인 검증 규칙·HITL(Human-in-the-Loop) 워크플로우로 자동화하여 회원 만족도와 심사관 생산성을 동시에 극대화하는 멤버십 핵심 서비스다.

1.1 사업 배경 (Why)

  • 수동 심사 한계: 멤버십 회원사가 제출한 수출입 계약서·외화매입증명·세금계산서 등 다양한 양식의 증빙서류를 심사관이 수동으로 검토 → 처리 지연·일관성 부족·심사관 1인당 처리량 한계 [RFP I-4-마, p.4]
  • 용역·전자적무체물 인정 어려움: 전통적 화물 수출과 달리 용역(서비스)·전자적무체물(디지털 콘텐츠)은 인정 범위·증빙 기준·분류 체계가 모호 → 신청자 혼란·반려율 상승 [RFP III-마-CSR-EC-001, p.74]
  • 위조·허위 신청 리스크: 외화매입증명·계약서 위·변조 가능성에 대한 시스템적 탐지 부재. 사고 발생 시 KITA 신뢰도 직격 [RFP III-마-SFR-EC-004, p.75]
  • 신청자 UX 미흡: 사용자가 어떤 서류가 필요한지·왜 반려되었는지를 직관적으로 알기 어려움. 신청 화면이 분산되어 있고, 진행 상태 알림이 부족 [RFP III-마-SFR-EC-001~005, p.74~76]
  • 데이터 사일로: 멤버십(KNEX)·수출실적증명·통합데이터플랫폼이 별도 운영되어 발급 데이터가 [포상]·기타 후속 서비스로 자연스럽게 흐르지 않음 [RFP III-마-INR-EC-001, p.77]

1.2 구축 목적 (What)

RFP I-4 추진범위 다섯째 항목 [RFP I-4, p.4]:

"멤버십 서비스의 용역 및 전자무체물 증명서 신청/발급 서비스의 회원 만족도 제고 및 업무 생산성을 극대화하기 위해 사용자 친화적 시스템 고도화 및 AI 기반의 증빙서류 인식 및 심사 시스템을 구축"

RFP I-4 명문 과제 내용:

"신청 내역 및 신청자 이력 기반 증빙서류 추천, 증빙서류 사전 검증, 실적증명 리포트 생성 기능 등 AI 기반 자동화된 실적심사 구축. 사용자 중심의 직관적인 신청/이력 관리, 가이드 체계 마련, 관리자 전용 화면 기능 개선을 통한 증명서 신청/발급 시스템 고도화"

1.3 최종 지향점 (Goal)

  • AI 자동심사 + HITL 결재: AI가 서류 인식·검증·승인 가능성·개선점을 제안하고, 심사관이 "검증표(일치/불일치/확인필요)" 기반으로 최종 결정 [RFP III-마-SFR-EC-005, p.76]
  • 서비스무역 특화 분류체계 표준화: 표준산업분류(KSIC)·한국은행 분류·대외기관 분류 간 연계 가능한 To-Be 분류체계 수립 + 시스템 기반 운영 [RFP III-마-CSR-EC-001, p.74]
  • Trade Agent 도메인 11종 중 1번: SFR-AP-009 [수출입 실적 증명 자동화] 전문 Agent와 강하게 결합. 본 시스템은 Agent의 호출 대상이자 결과 적재 대상 [RFP III-다-SFR-AP-009, p.57~58]
  • End-to-End 디지털 트레일: 신청·증빙등록·심사·승인·발급·후속 알림(카톡)까지 모든 행위가 감사 로그로 추적 가능

3. 핵심 요구사항 및 필수 고려사항 TOP 5

본 영역은 11개 요구사항으로 규모가 작지만, AI 자동심사 정확도·위조 탐지·HITL 결재·분류체계 표준화·내부 연계의 5축이 평가표 "기능 25 + 성능·품질 15 + 보안"의 직접 가산 포인트.

KEY-1. AI 자동인식·검증·우선순위 추천 (SFR-EC-005)

근거: [RFP III-마-SFR-EC-005, p.76] + [RFP I-4-마, p.4]

  • OCR 문서 뷰어: 업로드 서류(이미지/PDF)를 텍스트가 추출된 형태로 보여주는 뷰어
  • AI 분석 패널: 서류 + 입력 정보 분석하여 승인/반려 가능성 + 개선점 제안
  • AI 검증표: 사용자 입력값 vs AI 추출값을 일치/불일치/확인필요 3단 분류
  • 신청서 요약: 모든 정보를 한 곳에서 조회·수정 + 데이터 유효성 검사
  • FAQ 분석은 콜센터·TradePro 1:1 상담내역 연계 분석 (Trade Agent 연계)

난이도/배점 추정: ★★★★★

  • 본 시스템의 핵심 기능. 정확도 미달 시 자동심사 무력화 + 신뢰 사고
  • HWP/PDF 양식 다양성, 손글씨·도장·스캔 노이즈 등 한국 공공 문서의 한국어 OCR 난제
  • 평가위원이 "정확도 % + 처리시간 + 오탐지 시 책임 구조"를 직접 질문할 가능성 매우 높음

구현 핵심 포인트:

  • OCR: PaddleOCR 2.9 (한·영) + Docling(IBM, 표·차트·레이아웃 SOTA) + Tesseract 5 폴백
  • 레이아웃 이해: LayoutLMv3 (사전학습) + 도메인 fine-tuning (계약서·외화매입증명·세금계산서)
  • 필드 추출: 룰 기반(정규식·앵커) + LLM(Trade Agent SFR-AP-009 도메인 Agent) 이중
  • AI 판정: 일치/불일치/확인필요 3단 분류 (Confidence Score 표시) + Hallucination 통제(Ragas Faithfulness)
  • 심사 우선순위: 고위험(위조 가능성·신규 회원·금액 이상치) → 일반 → 저위험 자동 정렬
  • 산출물: 골든 데이터셋(서류 유형별 100건+) + 정확도 SLA(필드 추출 P95 ≥ 95%, 위조 탐지 Recall ≥ 90%)

KEY-2. 운영 체계·서비스무역 분류체계 표준화 (CSR-EC-001)

근거: [RFP III-마-CSR-EC-001, p.74]

  • AI 심사 기능 기반 내부 심사 규정 구체화 (To-Be 프로세스 적용 후 리스크 점검)
  • 민감정보 식별 기준 + 처리 방식 정의 (AI 심사 도입에 따른 신규 요구)
  • 서비스무역 산업 특화 분류체계 재수립 — 표준산업분류(KSIC) + 한국은행 분류 + 대외기관 분류 간 연계
  • 시스템 기반 To-Be 분류체계 운영 방안

난이도/배점 추정: ★★★★☆

  • 컨설팅 요구사항으로 평가표 "전략 및 방법론 15점"에 매핑
  • KITA + 대외 유관기관 합의가 필요한 영역 → 일정 리스크 큼
  • 분류체계 정의 지연 시 KEY-1의 검증 규칙도 동시 슬립

구현 핵심 포인트:

  • 분류체계 거버넌스: KITA + 한국은행 + 통계청 + 관세청 합동 협의체 사업 초기 1개월 내 발족
  • 매핑 테이블: KSIC → 한국은행 BOP → KITA 자체 분류 3단 매핑 시스템 구축
  • 운영 프로세스: AI 심사 기준·예외처리·HITL 결재 라인을 BPMN으로 표준화
  • 민감정보 처리: Microsoft Presidio + 한국어 PII (주민번호·사업자번호·외화 계좌)
  • AI 심사 규정: "AI는 추천, 결재는 사람" 원칙 명문화 (감사 대비)

KEY-3. 위조 서류 탐지 + 책임 동의 + 변경 로그 (SFR-EC-004)

근거: [RFP III-마-SFR-EC-004, p.75] + [SFR-EC-002 법적 책임 안내, p.74]

  • 서류 업로드 영역에 가이드 툴팁 + 위조 서류 제출 책임 동의 체크박스 (필수)
  • 수정신청 시 첨부 증빙파일 변경 내역 Log 표시
  • 사전안내 단계에서 허위/오입력 시 처벌 경고 + 관련 법령(대외무역법) 팝업

난이도/배점 추정: ★★★★☆

  • 위조 사고 발생 시 KITA 신뢰도 직격 + 형사 사건 가능성 → 보안 영역의 핵심
  • 평가표 "보안 요구사항"의 간접 매핑 + 발주처 잠재 두려움 1순위

구현 핵심 포인트:

  • 위조 탐지 기술 스택:
  • EXIF 분석: 작성/수정 시점 일치성, 카메라 메타데이터 검증
  • 이미지 해시: pHash·dHash 비교 (재사용 서류 탐지)
  • PDF 구조 분석: PDF 객체 트리 + 작성 도구 시그니처 분석
  • 이상 탐지: 폰트 일관성·여백·서명 위치 LayoutLM 기반 이상 점수
  • OCR 신뢰도 기반: 추출 문자의 비정상 패턴 탐지
  • 동의 UX: 모달 팝업 + 명시적 체크 + 타임스탬프 감사 로그
  • 변경 로그: 모든 파일 변경에 Append-only 로그(변경 전/후 해시·변경자·시각)
  • 워터마크: 발급 PDF에 KITA 디지털 서명 + 검증 가능 QR

5. 차별화된 제안 전략 (Value Proposition)

V-1. AI-First, Human-Final — "AI 추천, 사람 결재"의 책임 구조 명문화

고객 Pain: AI 자동심사 도입 시 "AI가 잘못 승인했을 때 누가 책임지는가"라는 거버넌스 공포가 가장 크다.

제안 해법:

  • "AI는 추천, 결재는 사람"을 시스템 아키텍처와 운영 규정으로 동시에 박음
  • 심사관 화면에 AI 판정·신뢰도·근거(SHAP)를 표시하지만 승인 버튼은 항상 사람
  • 모든 승인·반려에 심사관 ID·시각·AI 판정과 동의 여부를 감사 로그에 기록 (5년 보관)
  • AI 판정 임계치 이하(저신뢰)는 자동 HITL 강제, 임계치 이상은 심사관 1차 검토 후 결정
  • 차별화 메시지: "AI가 일을 빠르게 하지만 책임은 사람이 진다 — 감사 추적이 보장된다"

V-2. Forgery-Aware Pipeline — "위조 서류 탐지를 시스템에 박는다"

고객 Pain: 외화매입증명·계약서 위·변조 사고는 KITA의 잠재 사고 1순위. 그러나 RFP 명문 요구는 약하고 동의 체크박스 수준.

제안 해법:

  • EXIF·해시·PDF 구조·LayoutLM 이상 점수의 4중 위조 탐지 파이프라인
  • 위조 의심 점수 ≥ N% 시 심사 큐 최우선 + 심사관에게 "의심 영역 하이라이트"
  • 발급 PDF에 KISA 공인 PKI 디지털 서명 + QR 검증 (제3자가 진위 확인 가능)
  • 분기별 "위조 시도 통계" 발주처 공유 (탐지 성공 사례 + 신규 패턴)
  • 차별화 메시지: "신뢰 사고를 시스템 단계에서 봉쇄한다"

V-3. Service-Trade Classification Asset — "서비스무역 분류체계를 운영 자산으로 인계"

고객 Pain: 서비스무역·전자적무체물의 분류체계는 KITA에서도 명확히 정립되지 않음. 사업 종료 후 발주처가 혼자 운영해야 함.

제안 해법:

  • KITA + 한국은행 + 통계청 합동 협의체를 사업 초기 발족
  • KSIC ↔ BOP ↔ KITA 자체 분류 3중 매핑 테이블 + 시스템 마스터 데이터로 인계
  • 분류체계 변경 시 시스템 자동 동기화 + 영향 분석 리포트
  • AI 자동분류(KSIC 대분류) + 심사관 확인 UX
  • 차별화 메시지: "분류체계라는 운영 자산까지 인계받는다"

V-4. End-to-End Digital Trail — "신청부터 발급까지 모든 행위가 추적된다"

고객 Pain: 수동 심사 시대에는 누가·언제·왜 승인했는지 추적이 어려움. 감사 대응에 큰 부담.

제안 해법:

  • 신청자 입력·증빙 업로드·동의·AI 판정·심사관 결정·발급의 전 과정을 Append-only 로그
  • ClickHouse 기반 감사 대시보드 + 5년 보관 + 검색 (신청번호·회원사·심사관별)
  • 발급 PDF에 디지털 서명 + 발급번호 QR + 위변조 검증 페이지 공개
  • KITA 감사팀이 직접 로그 조회 가능한 운영 콘솔
  • 차별화 메시지: "감사·소송 대응이 시스템에서 즉시 가능하다"

V-5. Trade Agent 도메인 Agent와 일체형 — "11개 도메인 Agent 중 1번 선두"

고객 Pain: Trade Agent 27 Unit Agent 중 도메인 11종은 "정말 작동하는가"의 첫 검증 대상. 수출실적증명이 도미노의 첫 카드.

제안 해법:

  • Trade Agent SFR-AP-009 [수출입 실적 증명 자동화] 도메인 Agent와 본 시스템을 같은 PMO에서 통합 PoC
  • 발표 시 "회원사가 챗봇에 '내 수출실적증명 신청 도와줘'라고 물으면 본 시스템이 자동으로 호출되는" 시나리오 라이브 데모
  • 골든 데이터셋(서류 유형별 100건+) + Ragas 정량 임계치를 본 영역에서 먼저 검증
  • 도메인 Agent 11종 중 가장 먼저 제출되는 운영 자산 패키지의 표준 모델 제공
  • 차별화 메시지: "Trade Agent 도메인 깊이의 첫 증명"

7. 결론 및 제안 핵심 메시지

7.1 사업 본질 재정의

수출실적증명 자동화는 "기존 시스템 고도화" 사업이 아니라, "AI 자동심사 + HITL 결재 + 위조 탐지 + 분류체계 표준화"가 한 시스템으로 통합된 멤버십 핵심 신뢰 서비스의 디지털 전환이다. AI 정확도와 인간 결재 책임 구조의 양립이 본질적 도전.

7.2 제안 5축 메시지

  1. AI-First, Human-Final — AI 추천, 사람 결재의 명문화된 책임 구조
  2. Forgery-Aware Pipeline — 4중 위조 탐지 + 디지털 서명 + 검증 QR
  3. Service-Trade Classification Asset — 분류체계까지 운영 자산으로 인계
  4. End-to-End Digital Trail — 5년 감사 추적 + 검색 가능한 로그
  5. Trade Agent 도메인 깊이의 첫 증명 — 11개 도메인 Agent 중 1번 선두
