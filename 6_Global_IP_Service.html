1. 목표 시스템 정의

1-Liner Definition

글로벌 지재권 특화정보서비스는 수출 중소기업의 지재권 생애주기(아이디어 구상 → 개발 → 보호 → 사업화) 전 단계에 걸쳐 KITA·한국지식재산보호원·한국발명진흥회·KIPRIS(특허청)·한국콘텐츠진흥원 등 분산된 지재권 데이터·서비스·교육·비용지원·분쟁대응을 단일 포털과 AI 기반 개인화 모니터링 위젯으로 통합 제공하는 외부 기관 연계 중심의 신규 도메인 서비스다.

1.1 사업 배경 (Why)

  • 지재권 정보의 분산성: 한국지식재산보호원·한국발명진흥회·KIPRIS·콘텐츠진흥원·특허법인·외부 컨설팅사 등에 정보·서비스·지원사업이 분산되어 있어 수출 중소기업이 "어디에서 무엇을 받을 수 있는지" 파악 자체가 어려움 [RFP I-4-바, p.4]
  • 실시간 위험 모니터링 부재: 특허분쟁 위험등급, 경쟁사 특허 활동, 무단선점 피해 등 지재권 시그널이 회원사 단위로 통합 모니터링되지 않음. 사후 대응 중심 → 사고 예방 한계 [RFP III-바-SFR-IP-004, p.81]
  • 출원·검색의 진입 장벽: KIPRIS 등 외부 검색 도구를 회원사가 직접 사용해야 하며, 결과 데이터가 KITA 플랫폼에 적재·재활용되지 않음. 검색 → 출원 → 사업화로 이어지는 동선 단절 [RFP III-바-SFR-IP-010, p.83]
  • 비용 지원 사업의 정보 비대칭: 한국발명진흥회·한국지식재산보호원의 비용 지원 사업이 다수 존재하지만 회원사가 자신에게 적합한 사업을 찾고 신청하기 어려움. KITA 산업 참여 실적 기반 가점이 자동 산출되지 않음 [RFP III-바-SFR-IP-006, p.82]
  • 회원사 콘텐츠IP ↔ 유체물 제조사 협업 부재: 콘텐츠IP 보유 회원사와 유체물 제조사·유통사 간 협업 매칭 채널이 없음 [RFP III-바-SFR-IP-005, p.81]
  • AI 상담 학습 데이터 부족: TradePro AI 상담의 지재권 분야 답변 정확도가 학습 데이터 부족으로 한계 [RFP III-바-SFR-IP-007, p.82]

1.2 구축 목적 (What)

RFP I-4 추진범위 여섯째 항목 [RFP I-4, p.4]:

"수출 기업의 지재권 전략적 활용 중요성이 확대됨에 따라 지재권 확보/관리/활용을 분산된 정보를 통합하여 체계적으로 지원할 수 있는 특화 서비스 구축. 지재권 관련 유관 기관 및 전문 기관 연계, 기관별 역할 및 협력 범위 수립, 상호 서비스 운영 방안을 반영한 협업구조 설계"

CSR-IP-003 명문 [RFP III-바-CSR-IP-003, p.79]:

"지재권 생애주기(아이디어 구상-개발-보호-사업화) 기반 서비스 제공 구조 및 시스템 아키텍처 정의. 사용자 여정 기반 서비스 탐색 및 진입 경로 설계"

1.3 최종 지향점 (Goal)

  • One-Stop Lifecycle Portal: 지재권 4단 라이프사이클(아이디어 → 개발 → 보호 → 사업화)에 따른 사용자 여정 기반 통합 진입점 + 단계별 KITA 자체 + 외부 기관 서비스 자동 노출
  • AI-Driven Risk Monitoring: 회원사 관심정보 기반 특허분쟁 위험등급·경쟁사 특허 활동·제도 변경을 시각화 + 카톡 채널·이메일 자동 알림 [RFP III-바-SFR-IP-004, p.81]
  • External Org Federation: 한국지식재산보호원·한국발명진흥회·KIPRIS·콘텐츠진흥원과의 협업 거버넌스(R&R, End-to-End 프로세스, 데이터 연계 정책)를 시스템에 박은 표준 연합 모델
  • Membership Multiplier: KIPRIS 검색 결과를 회원사 DB에 적재 → 추천·학습·분쟁 대응으로 재활용. 비용 지원 사업 가점 자동 산출 → 멤버십 가치 강화

3. 핵심 요구사항 및 필수 고려사항 TOP 5

본 영역은 14개 요구사항으로 도메인 11 Agent 중 가장 외부 기관 의존도가 높음. AI 위험 모니터링·KIPRIS 임베딩·비용 지원 매칭·외부 기관 거버넌스·TradePro AI 상담 강화가 평가 핵심.

KEY-1. AI 기반 지재권 모니터링 위젯 (SFR-IP-004)

근거: [RFP III-바-SFR-IP-004, p.81]

  • MyPage 등록 고객 정보(관심 산업·품목·국가) 기반 지재권 관련 모니터링 정보 제공
  • 외부 지식재산 데이터(IP Navi 등) + 산업 정보 활용 → 특허분쟁 위험등급, 경쟁사 특허 활동, 무단선점 피해현황 등 주요 지재권 이벤트 정보를 대시보드 형태로 시각화
  • 지재권 위험 등급 변화·경쟁사 활동 변화·제도 변경 등 주요 이벤트 발생 시 카톡 채널 및 이메일을 통해 알림 제공
  • AI 기반 맞춤 서비스 추천 + AI 기반 맞춤 정보 탐색(산업 분석 보고서 자동 생성)

난이도/배점 추정: ★★★★★

  • 본 영역의 대표 기능. SFR-IP-004는 14개 요구사항 중 정보량과 복잡도가 가장 높음
  • AI 위험등급 판정의 정확도·책임 구조 + 한국지식재산보호원 IP Navi/IP Alert 연계가 동시 변수
  • 평가표 "기술 및 기능 25점"의 직접 가산 영역

구현 핵심 포인트:

  • 위험등급 모델: 특허청 분쟁 데이터 + 회원사 산업/HS 매칭 + 경쟁사 특허 출원 패턴 기반 ML(XGBoost/LightGBM) 분류
  • 외부 데이터 수집: 한국지식재산보호원 IP Navi/IP Alert API 사용 협약 + 일/주 단위 동기화
  • 알림 트리거: 위험등급 변화(High↑/Mid→High), 경쟁사 신규 출원, 제도 변경 등 ~15개 트리거 표준화
  • 시각화: 게이지(위험등급) + 타임라인(경쟁사 활동) + 카드(이벤트) 3중 위젯
  • 보고서 생성: Trade Agent SFR-AP-009 도메인 Agent 호출 → 산업 분석 보고서 자동 생성 (분기·요청 시)

KEY-2. KIPRIS 임베딩 + 검색결과 적재 + 후속 동선 (SFR-IP-010)

근거: [RFP III-바-SFR-IP-010, p.83]

  • IP 검색 대상 국가 기반 검색 경로 안내
  • KIPRIS 검색 화면 임베딩 제공 (사용자가 별도 외부 이동 없이 플랫폼 내에서 검색 수행)
  • KIPRIS 검색 결과를 무역협회 플랫폼에 연계하여 회원사 DB로 적재 → 맞춤 서비스 추천 등 후속 활용
  • 선행기술조사 교육 콘텐츠 연계 (무역아카데미)
  • KIPRIS 검색이 어려운 국가 정보 필요 회원사는 멤버십 제휴 특허 검색 서비스 할인 페이지로 연계

난이도/배점 추정: ★★★★★

  • KIPRIS 임베딩은 특허청과의 별도 연계 협약·정책 검토 필요. SI사 단독으로는 해결 불가
  • 검색 결과 적재의 데이터 권한·저작권·개인정보 이슈 동시 발생
  • 평가위원이 "특허청과 사전 협의 진행 상태"를 직접 질문할 가능성 높음

구현 핵심 포인트:

  • KIPRIS 임베딩: iframe + sandbox 격리 + CSP/CORS 정책 협의 (특허청 KIPRIS 정책 사전 확인)
  • 결과 적재: 검색식 + 결과 메타(특허번호·출원인·요약) → KITA Lakehouse Bronze → 회원사별 검색 이력 DB
  • 권한 분리: 비회원은 검색만, 회원은 적재 + 추천, 멤버십 회원은 추가 할인 연계
  • 교육 연계: KIPRIS 사용 중 "선행조사 방법" 학습 버튼 → 무역아카데미 Deeplink + 진도 추적
  • 멤버십 제휴: 외부 검색 서비스(예: WIPS, Patsnap 등)와 할인 협약 + Deeplink 연계
  • 폴백: KIPRIS 임베딩 정책 변경 시 외부 이동 모드 자동 전환

KEY-3. 외부 기관 거버넌스 (CSR-IP-001/002)

근거: [RFP III-바-CSR-IP-001, p.79] + [CSR-IP-002, p.79]

  • 지재권 특화 서비스 운영체계 수립 (포트폴리오·분류체계·정책·KPI)
  • 외부 전문기관 협력 체계 구축: 협력 대상 선정·공동 서비스 모델·역할 분담·R&R·E2E 프로세스·이슈 관리
  • 데이터 관리 및 연계 체계 설계: 데이터 유형·기관별 활용 가능 데이터·연계 방식(API)·수집 주기·갱신 정책
  • 민감정보 식별 + 마스킹 처리 (음성/Text/이미지 등 비/정형 데이터 내 식별 정보)

난이도/배점 추정: ★★★★☆

  • 컨설팅 요구사항 — 평가표 "전략 및 방법론 15점" 직접 매핑
  • 한국지식재산보호원·발명진흥회·KIPRIS·콘텐츠진흥원 4개 기관과 동시 협의 → 행정 일정 슬립이 가장 큰 리스크
  • 컨설팅 부분이 부실하면 KEY-1, KEY-2의 기술 구현도 동시 슬립

구현 핵심 포인트:

  • 외부 기관 협의체: KITA + 4개 기관 합동 거버넌스 협의체 사업 초기 1개월 내 발족 (발주처 행정 지원 사전 합의 필수)
  • 협업 모델: API 직접 연계 / 데이터 교류 / 신청 Deeplink / 콘텐츠 임베딩 등 기관별 차등 모델 설계
  • 데이터 정합성·최신성 SLA: 기관별 데이터 갱신 주기(실시간/일/주/월) 명시 + 정합성 검증 자동화
  • 민감정보: Microsoft Presidio + 한국어 PII + 기업정보(사업자번호) 마스킹 + 마스킹 전 원본 분리 보관
  • 기관별 Contact Point + 이슈 대응 SLA 명시

KEY-4. 비용 지원 사업 매칭 + 가점 자동 산출 (SFR-IP-006)

근거: [RFP III-바-SFR-IP-006, p.81~82]

  • 외부 기관(한국발명진흥회·한국지식재산보호원 등) 비용 지원 사업 정보 연계 + 조회
  • IP 업무 단계별 지원 사업 리스트 + 지원 대상·내용 표시
  • MyPage 기업 정보(규모·산업) 및 서비스 이용 정보 기반 신청 가능 사업 선별
  • 한국무역협회 산업 참여 실적 기반 지원 사업 신청 시 적용 가능한 가점 자동 산출
  • 외부 기관 신청 화면 Deeplink

난이도/배점 추정: ★★★★☆

  • 회원사 입장의 가장 직접적 ROI 기능 — "지원금 N백만원 받았다"는 정량 성과
  • 가점 자동 산출은 KITA 산업 참여 실적 데이터 정합성에 의존 (TDB 의존)

구현 핵심 포인트:

  • 사업 정보 수집: 발명진흥회·보호원 API or 웹 스크래핑(주 1회) → IP Master DB 적재
  • 매칭 엔진: 사업 자격 조건(매출·업력·지역·산업) vs MyPage 정보 룰 매칭
  • 가점 산출: KITA 멤버십 등급 + 산업 참여 실적(전시회·교육·상담 횟수) → 사업별 가점 룰 적용
  • 신청 흐름: 매칭 결과 → 회원사 확인 → 외부 신청 Deeplink + 신청 진행 상태 알림
  • 사후 추적: 신청 완료 → 결과(승인/반려) 수동 또는 외부 데이터 동기화 → KITA 데이터에 반영

KEY-5. TradePro AI 상담 지재권 RAG 강화 (SFR-IP-007)

근거: [RFP III-바-SFR-IP-007, p.82] + [RFP III-다-SFR-AP-009 지재권 도메인 Agent, p.57~58]

  • 기존 TradePro AI 상담 서비스에 외부 자료 + 참조 데이터 추가 → 답변 정확도 향상 + 답변 가능 분야 확대
  • 내부: 무역협회 지재권 세미나 발표 자료, TradePro 1:1 상담 내용 자료
  • 외부: 지재권 관련 뉴스, 지식재산처 법/제도/동향, 레포트
  • 원천 데이터 조회 + 다운로드 기능 (출처 인용 + 신뢰도 확보)
  • 상담 내용 + 사용자 질문 정보 기반 관련 서비스 추천

난이도/배점 추정: ★★★★☆

  • Trade Agent의 지재권 도메인 Agent와 직결 (도메인 11 중 하나의 핵심)
  • 출처 인용·다운로드 기능으로 Hallucination 통제 정조준

구현 핵심 포인트:

  • 학습 데이터 정제: TradePro 상담 텍스트 → PII 마스킹(Presidio) → 청킹 → 임베딩(KURE-v1)
  • 외부 데이터: 뉴스(크롤링) + 지식재산처 정책 자료(공식 API or 수집) + 자동 갱신 파이프라인
  • RAG: Milvus + BGE-Reranker → 답변 + 출처 N건 인용 + 다운로드 링크
  • 신뢰도: Ragas Faithfulness + Context Precision 임계치 기반 응답 차단
  • Trade Agent 통합: SFR-AP-009 지재권 도메인 Agent의 일부 기능으로 통합 (LiteLLM Proxy 경유)

5. 차별화된 제안 전략 (Value Proposition)

V-1. Lifecycle-Native UX — "지재권 4단 라이프사이클이 화면 골격이 된다"

고객 Pain: 분산된 지재권 정보를 단순 통합하면 또 다른 메뉴 트리가 됨. 회원사 입장에서 "내가 지금 어느 단계에 있고 다음에 무엇을 해야 하는지"가 보이지 않음.

제안 해법:

  • 모든 화면의 IA(정보 구조)를 "아이디어 → 개발 → 보호 → 사업화" 4단으로 통일
  • 회원사 행동 데이터로 자동 단계 추정 + 단계별 다음 액션 추천
  • 단계별 KPI(예: 보호 단계의 출원 건수·등록률) 위젯 자동 노출
  • 차별화 메시지: "지재권 전략을 화면이 가르쳐준다"

V-2. External Federation Pattern — "4개 기관을 단일 PMO에서 묶어낸다"

고객 Pain: 외부 기관 4곳과 동시 협의는 단독 SI사가 가장 두려워하는 영역. 행정 일정 슬립 가능성 매우 큼.

제안 해법:

  • 사업 초기 1개월 내 4개 기관 합동 거버넌스 협의체 발족 (KITA + 보호원 + 진흥회 + KIPRIS + 콘진원)
  • 기관별 협업 모델 매트릭스(API/데이터 교류/Deeplink/임베딩) 표준화
  • 협의 일정 슬립 시 우선 모킹 인터페이스로 개발 병행 → 협약 후 실연계 스왑
  • 기관별 SLA + 이슈 대응 절차를 운영 자산으로 인계
  • 차별화 메시지: "외부 기관 거버넌스를 sicht에 묶어낼 수 있는 통합 SI"

V-3. Risk-as-a-Service — "지재권 위험을 회원사 화면으로 매일 보낸다"

고객 Pain: 한국지식재산보호원 IP Alert는 존재하지만 회원사가 직접 가입·구독해야 활용. KITA에서 통합 푸시되는 채널이 없음.

제안 해법:

  • 회원사 관심정보 자동 등록 → 위험등급·경쟁사 활동·제도 변경 매일 모니터링 → 카톡 알림톡 + 이메일
  • 위젯에서 "왜 위험등급이 올랐는가" Explain 텍스트 + 추천 액션(상담·교육·비용 지원) CTA
  • 분기별 회원사별 "지재권 리스크 리포트" 자동 생성 + 다운로드
  • 차별화 메시지: "지재권 사고를 사후 처리에서 사전 예방으로"

V-4. Cost-Support Concierge — "지원금을 시스템이 찾아준다"

고객 Pain: 한국발명진흥회·보호원의 다수 비용 지원 사업이 있으나 회원사가 신청 가능한 사업을 찾고 가점을 계산하는 것이 어려움.

제안 해법:

  • MyPage 기업 정보 + KITA 산업 참여 실적 → 신청 가능 사업 자동 매칭 + 가점 자동 산출
  • 사업 마감 D-N 알림 + 신청 진행 상태 추적
  • 분기별 매칭 사업 리스트 보고서 + 회원사별 "이번 분기 받을 수 있는 지원금" 정량 표시
  • 차별화 메시지: "회원사가 받을 수 있는 지원금을 시스템이 먼저 찾아준다"

V-5. KIPRIS-Embedded Search Asset — "검색 한 번이 KITA 자산이 된다"

고객 Pain: KIPRIS 검색 결과는 회원사가 사용하고 끝남. KITA 입장에서는 "회원사가 무엇을 검색했는지·왜 검색했는지" 데이터가 전혀 축적되지 않음.

제안 해법:

  • KIPRIS 임베딩 + 검색식·결과 메타를 회원사 동의 하에 KITA Lakehouse에 적재

7. 결론 및 제안 핵심 메시지

7.1 사업 본질 재정의

글로벌 지재권 특화정보서비스는 "지재권 정보 포털 구축"이 아니라, "4개 외부 기관(보호원·진흥회·KIPRIS·콘진원) 거버넌스 + 지재권 라이프사이클 UX + AI 위험 모니터링 + 비용 지원 매칭"이 한 시스템으로 통합된 외부 연합 중심 신규 도메인 서비스다. 단독 SI 기능 구현이 아니라 외부 기관 PMO를 끌고 갈 수 있는 통합력이 본질.

7.2 제안 5축 메시지

  1. Lifecycle-Native UX — 4단 라이프사이클이 화면 골격이 되는 통합 진입점
  2. External Federation Pattern — 4개 외부 기관을 단일 PMO에서 묶어내는 통합 SI 역량
  3. Risk-as-a-Service — 지재권 사고를 사후에서 사전 예방으로 전환
  4. Cost-Support Concierge — 지원금을 시스템이 회원사 대신 찾아준다
  5. KIPRIS-Embedded Search Asset — 회원사 검색 한 번이 KITA의 추천 자산이 된다
