차세대 AX무역플랫폼 구축 사업
일반현황 및 관리요건 요약

제안팀이 사업 성격, 일정, 제출 제약, 평가 포인트를 즉시 파악할 수 있도록 RFP와 ISP의 핵심 관리 정보를 정리한 보고서

사업명차세대 AX무역플랫폼 구축 사업
발주기관한국무역협회 무역플랫폼실

핵심 방향AI 서비스 전환 + 원스톱 수출지원 + 클라우드 인프라

1. 사업 개요

구분내용제안 PM 관점
사업명 차세대 AX무역플랫폼 구축 사업 단순 포털 개편이 아니라 데이터·AI·업무서비스·인프라를 묶은 전사 플랫폼 전환 사업
발주기관 한국무역협회 무역플랫폼실 무역진흥 선도기관의 디지털 서비스 허브 재구축 성격
사업기간 계약 체결일로부터 12개월 이내 8개 이상 주요 구축 영역을 12개월 내 병렬 수행해야 하므로 통합 PMO와 단계별 품질 게이트가 중요
예산/규모 RFP 초안 내 구체 금액 미기재 가격제안서는 총합계금액과 항목별 금액을 부가세 포함 기준으로 상세 제출 필요
주요 범위 Trade Digital Brain, Trade Agent, 통합대시보드, 수출실적증명, 글로벌 IP, tradeKorea, ESG, 무역아카데미, 클라우드 공통 인프라 데이터 플랫폼이 중심축이고, 각 업무 서비스는 AI·개인화·연계 기반으로 고도화
평가 구조 전략/방법론 15점, 기술/기능 25점, 성능/품질 15점, 프로젝트 관리 15점, 프로젝트 지원 20점, 조직/인력 10점 기술 구현안뿐 아니라 관리·품질·운영·인력 증빙이 수주 핵심
제안서 첫 장의 포지셔닝은 “AI 기능 구축”보다 무역 전주기 업무를 데이터·AI·클라우드 기반으로 연결하는 AX 플랫폼 전환으로 잡는 것이 적합하다.

2. 추진 배경 및 필요성

고객사가 사업을 시작한 이유

  • 국내 수출기업이 요구하는 지능형 수출지원 서비스 제공을 통해 무역진흥 선도기관 위상 강화 필요
  • 공공·민간 데이터 연계와 통합을 통해 KITA 플랫폼을 무역 생태계 허브로 전환할 필요
  • 사용자 여정의 전주기를 지원하는 원스톱 업무서비스 요구 증가
  • 무역 업무에 필요한 정보가 여러 서비스와 메뉴에 분산되어 있어 고객이 필요한 정보를 적시에 찾기 어려움
  • 단순 검색·상담 중심 서비스에서 AI Agent 기반 업무 지원으로 전환해야 하는 시점
  • 클라우드 기반 확장성과 보안 운영체계를 확보해 인프라 노후화와 성능 이슈를 해소해야 함

핵심 Pain Point

사용자 경험

  • 복잡한 메뉴 구조와 분산된 진입 채널로 접근성 저하
  • 검색 정확도 부족으로 정보 탐색 효율 저하
  • 타 서비스 이동 시 기존 입력 데이터 재입력 발생

데이터/AI 활용

  • 서비스 간 정보 단절 및 데이터 미수집으로 고도화 한계
  • AI가 즉시 활용 가능한 데이터 정보/구조 부족
  • 전사 통합 데이터 품질관리 기준 부재

업무 생산성

  • 상담 대기, 바이어/셀러 매칭, 증빙 검증 등 반복·수작업 업무 부담
  • 수작업 처리로 정확도 저하 및 처리 지연 리스크 존재
  • 업무별 지원 서비스가 단절되어 E2E 업무 완결성이 부족

운영/인프라

  • IT 확장성 부족 및 인프라 노후화에 따른 성능·속도 이슈
  • AX 전략 실행을 위한 전담 체계와 단계적 목표 수립 미흡
  • 서비스별 데이터 품질과 운영 수준 편차 존재

3. 사업의 목적

정성 목표

  • 글로벌 무역 데이터 플랫폼 기반의 무역 특화 지능형 Trade Agent 구축
  • 분산 데이터의 통합 View 제공 및 AI Agent 연계를 통한 맞춤형 대시보드·알림 제공
  • 수출 준비부터 글로벌 디지털 무역 연계, 사후관리까지 원스톱 E2E 무역 수출 업무 지원 구현
  • tradeKorea, 수출실적증명, 무역아카데미의 AI 도입 및 기능 고도화
  • ESG, IP 신규 플랫폼 구축을 통한 글로벌 수출지원 서비스 확대
  • 클라우드 랜딩존, 공통 운영환경, 보안체계 구축을 통한 인프라 고가용성·확장성 확보

정량/관리 목표

  • 사업기간: 계약 체결일로부터 12개월 이내
  • 공통 요구사항: 40개, 전체 요구사항: 181개
  • Trade Agent: 범용 기능 Agent 16종 및 도메인 특화 Agent 11종 개발 요구
  • ISP 기준 Agent Use Case: 88개 도출, Agent 기능 후보 56개 식별
  • 평가 배점상 기술/기능 25점, 프로젝트 지원 20점, 전략/방법론·성능품질·프로젝트관리 각 15점

5. 제안 시 필수 고려사항

입찰·참가 및 인력 관련

  • 참여 예정 인력은 개인별 세부 이력서와 수행실적을 제출해야 함
  • 사업자 선정 후 참여인력의 이력 증빙, 예: 고용보험 납입 내역 등을 제출해야 함
  • 타 과제 참여 중인 인력을 개발인력으로 제안하면 기술평가 대상 제외, 선정 후 확인 시 사업자 선정 무효 가능
  • 공동수급 가능 양식이 포함되어 있으며, 공동수급 시 대표자 및 공동수급현황표, 지분율, 책임 범위를 명확히 해야 함
  • 하도급은 발주처 승인 시에만 가능하며, 관련 증빙 제출 필요

제안서 작성 제약

  • 모호한 표현, 예: “할 수 있다”, “할 예정이다”, “고려하고 있다”는 불가능으로 간주
  • 허위 또는 예상 기반 기재 금지, 모든 기재사항은 객관적 입증 가능해야 함
  • 제안서 목차는 제안 개요, 업체 일반사항, 용역수행계획, 수행 일정, 수행 방법, 조직·업무분장, 인력 이력 중심으로 구성
  • 최근 3년간 관련 유사용역 수행실적 제시 및 실적증명원 원본 제출 필요
  • 소프트웨어 라이선스는 정품 원본 기준, 공급사 인증서 등 증빙자료 제출 요구 가능

제출 방식 및 서류

  • 제안서는 직접 제출해야 하며 우편접수 불가
  • 제안서 제출 시 관련 파일을 담당자에게 이메일 송부 필수
  • 제안서는 공문 첨부 및 대표자 인감 날인 필요, 날인 없을 경우 제안으로 인정하지 않음
  • 서식 기준 제출물: 용역제안서 제출신청서, 제안서 10부, 제출서류 목록 및 관련서류 1부
  • 가격제안서는 총합계금액과 항목별 금액을 자세히 작성하고 부가가치세 포함 기준으로 제출
  • 제안 설명회 개최 시 실제 사업에 참여하는 PM이 발표해야 함

관리·수행 요건

  • 분석, 설계, 개발, 테스트, 전환 단계별 일정과 산출물, 선·후행 관계를 구체화해야 함
  • 범위, 일정, 품질, 위험, 변경사항에 대한 관리체계와 절차 제시 필요
  • 착수, 정기, 중간, 완료 및 수시보고 체계를 포함해야 함
  • 산출물 작성·관리·제출 방안과 형상·문서관리 체계 제시 필요
  • 서비스 전환 일정은 발주처와 협의하되, 주요 업무 일정 영향 최소화와 장애 복구 절차 확보 필요
  • 보안서약, 보안교육, 자료·장비·산출물 보안관리 및 하도급 동일 보안 기준 적용 필요

6. 핵심 관전 포인트

AI Agent Data Lakehouse E2E 수출지원 초개인화 대시보드 Cloud Landing Zone PMO·품질·보안

수주를 위해 제안서에서 강조해야 할 6가지

  • AI Agent 실현 가능성: Master/Orchestration/Unit Agent 구조, 평가 지표, 운영 모니터링, Hallucination 관리까지 구체화
  • AI-Readable Data Foundation: 분산 데이터를 Lakehouse와 AI Data Hub로 통합하고, 메타데이터·품질·리니지·보안 체계를 함께 제시
  • 무역 전주기 원스톱 업무 완결성: 준비, 정보탐색, 바이어 매칭, 증빙, ESG/IP, 교육, 사후관리까지 연결되는 사용자 여정 중심 제안
  • 개인화·알림 기반 고객 체감: 회원정보, 행동로그, 관심국가/품목 기반 대시보드와 선제적 알림으로 “매일 쓰는 플랫폼” 메시지 강화
  • 클라우드·보안·성능 운영모델: Landing Zone, 이중화, 모니터링, 백업/복구, 비용관리, 보안관제, 성능 테스트까지 운영 가능한 구조로 제시
  • 12개월 병렬 구축 PMO 역량: 181개 요구사항, 다수 도메인, 데이터/AI/인프라 동시 추진을 통제할 일정·위험·품질·산출물 관리체계 강조
제안 메시지는 “기능을 많이 만든다”가 아니라 무역협회가 데이터와 AI로 회원사의 수출 업무를 24시간 지원하는 운영 플랫폼을 갖게 된다로 정리하는 것이 가장 선명하다.
