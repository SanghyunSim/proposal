1. 사업 본질 — 한 줄 정의

KITA가 운영 중인 8개 분산 서비스를 단일 통합 데이터 기반(Lakehouse) 위에서 재구성하고, 그 위에 Multi-Agent 기반 27종 Trade Agent와 초개인화 통합대시보드를 올려 수출기업 업무 전주기를 단일 UX로 제공하는 12개월짜리 통합 SI 사업. AI 챗봇 도입 사업이 아니라 "데이터 통합 + AI Agent + 8개 서비스 동시 신·증축 + 클라우드 랜딩존 신규 구성"이 한 계약 안에 묶인 멀티트랙 SI가 본질.

왜 이 사업인가 (As-Is 문제 정의)

  • 서비스 사일로화 — KITA.net·tradeKorea·TradePro·무역통계·무역아카데미·멤버십·수출실적증명·KNEX(내부)가 별도 운영. 회원 식별·행동 로그·콘텐츠 메타가 서비스 경계에 갇혀 있음.
  • 데이터 활용 한계 — 정형/비정형/행동 데이터의 표준·메타·리니지·라이프사이클이 부재. AI 학습/추론에 바로 쓸 수 있는 "AI Readable Data" 계층이 없음.
  • AI 부재 또는 단발성 — TradePro 일부에만 AI 상담 존재. HS Code 매칭, 바이어 추천, 서류 자동심사, ESG 진단, 지재권 모니터링 등 도메인 AI 시나리오 미구현.
  • 인프라 노후 및 신규 수요 미대응 — 무역통계 운영계는 Solaris 11/Oracle 19c/NetApp 기반 온프레, ESG·지재권은 시스템 자체가 없음. 클라우드 랜딩존·MCP 서버 신규 구축이 동시에 필요.

RFP 명문 사업 목표 (Section I-3)

  1. 글로벌 무역 데이터 플랫폼 기반 무역 특화 지능형 Trade Agent 구축
  2. 수출 기업의 업무 전주기를 지원하는 원스톱 통합 서비스 도입 (tradeKorea·수출실적증명·무역아카데미 AI 도입·고도화 + ESG·IP 신규)
  3. 클라우드 환경 구성을 통한 인프라 운영의 확장성·민첩성 확보 (랜딩존·공통 운영환경·보안체계)

3. 시스템 아키텍처 — 5계층 구조

전체 산출물은 다음 5계층으로 정렬된다. 하위 계층의 산출물이 상위 계층의 전제조건이며, 임계 경로(Critical Path)는 L2 → L3 → L5 순서다.

L5UX / Agent
Trade Agent (Multi-Agent) + 단일 포털 UI
사용자가 직접 상호작용하는 계층. Master Agent가 의도·맥락 분석 후 Orchestration이 Unit Agent 호출 사슬을 구성. 모든 Agent는 표준 아키텍처(시스템 프롬프트·지식데이터·파이프라인) 기반으로 구현되며, LLM 추상화(LiteLLM)로 모델 종속을 회피.
구성요소: Master Agent · Orchestration · Unit Agent 27종 · Agent Portal Web · SSO · 운영자 콘솔(LLM/파라미터 관리)
L4Personalization
통합대시보드 + 알림·추천
사용자 회원정보·행동로그·관심도(HS·국가·업종)를 기반으로 위젯 자동구성. AI Agent의 추천 결과를 화면 위에 띄우고, 다채널 알림으로 푸시.
구성요소: Dashboard Engine · Recommendation Engine · Notification Hub (SMS/메신저/App Push) · A/B 위젯 관리
L3Domain Services
8개 도메인 서비스 (신규 4 / 고도화·전환 4)
실제 비즈니스 트랜잭션이 발생하는 계층. 모든 서비스는 L2 데이터플랫폼과 표준 API/이벤트로 연계되며, L5 Agent가 도메인 작업을 수행할 때 MCP 서버를 통해 호출됨.
구성요소: tradeKorea · 수출실적증명 · 무역아카데미 · 무역통계(전환) · ESG 플랫폼 · 지재권 서비스 · TradePro AI 고도화 · 통합 회원/SSO
L2Data Foundation
Trade Digital Brain (Data Lakehouse + AI Data Hub)
전사 데이터의 단일 진실 공급원(SSOT). 정형/비정형/반정형 수집 → Parsing/Chunking/Embedding → Lakehouse 적재 → 메타·표준·품질·리니지·라이프사이클 거버넌스. AI Data Hub는 벡터·지식베이스·프롬프트 라이브러리를 표준 메타 기반으로 관리.
구성요소: Lakehouse (ACID·메타·고속분석) · AI Data Hub (Vector·RAG) · ETL/CDC 파이프라인 · 메타데이터 카탈로그 · DB 모델링 · DQ/리니지 · APIM
L1Infrastructure
클라우드 랜딩존 + 보안 + MCP 인프라
멀티 Account 랜딩존, 전용회선, 모니터링/로그/백업/비용관리, 보안체계, 고가용성/Auto Scaling. AI Agent 연계용 MCP 서버를 협회 도메인별로 신규 구축.
구성요소: 클라우드 랜딩존 · 네트워크/보안 · 통합 모니터링 · 로그/APM · 백업/DR · MCP 서버 (도메인별)

4. 시스템 간 의존관계 — 임계 경로 식별

제안 일정 설계 시 반드시 인지해야 할 핵심 의존성. L2 Trade Digital Brain이 모든 상위 계층의 전제조건이며, L2가 늦으면 L3/L4/L5 전체가 일정 슬립.

L5 Trade Agent→L2 Digital Brain
Agent의 RAG·지식베이스·프롬프트 라이브러리·메타데이터가 L2에 표준화되어야 응답 품질 보장. AI Data Hub 미완성 시 Agent는 빈 깡통.
L5 Trade Agent→L3 도메인 서비스
Agent의 Tool Use(MCP) 호출 대상이 L3. 수출실적증명 심사·tradeKorea 매칭·ESG 진단을 Agent가 직접 수행하려면 L3 API가 MCP로 노출되어야 함.
L4 대시보드→L2 + L5
L2의 사용자 행동로그·관심 메타로 위젯 구성, L5 추천 Agent의 결과를 위젯에 노출. 양쪽 모두 의존.
L3 도메인 서비스→L2 Digital Brain
서비스 간 데이터 공유는 L2를 경유해야 함 (Point-to-Point 연계 금지). 무역통계 전환은 L2 Lakehouse로 직접 마이그레이션.
전 계층→L1 인프라
랜딩존·SSO·보안·MCP 서버가 선행되지 않으면 어떤 계층도 구동 불가. L1은 사업 초기 1~2개월 내 완료가 필수.
전 계층→통합 SSO/IAM
단일 세션 유지·RBAC·SSO가 L1의 인증 컴포넌트로 일원화되어야 함. 8개 서비스 + 대시보드 + Agent 포털이 같은 토큰 체계 사용.

임계 경로 (Critical Path)

M1 L1 랜딩존·MCP 베이스 → M2 L2 Lakehouse·AI Data Hub MVP → M3 L3 도메인 API/MCP 서버 노출 → M4 L5 Master/Orchestration + Unit Agent 27종 단계 오픈 → M5 L4 대시보드 위젯 + 알림 → M6 통합 테스트·전환 리허설. 병렬 가능 트랙은 L3 8개 도메인 서비스 내부 고도화이지만, MCP·메타데이터 연계 시점은 M3에 동기화 필요.

5. 엔드투엔드 시나리오 — 5계층이 한 요청에 어떻게 맞물리는가

예시: 회원사 사용자가 "베트남에 우리 화장품 수출하려는데 절차와 바이어를 알려달라"라고 요청한 경우.

사용자가 통합 포털 접속. SSO 토큰 발급(KITA 통합 IAM) 후 세션 생성.
L1 인프라 / L3 SSO
대시보드가 사용자 프로필·행동로그·HS Code·관심국가를 조회해 위젯 자동구성. 동시에 추천 Agent 호출.
L4 대시보드 ↔ L2 Digital Brain
사용자 자연어 입력 → Master Agent가 의도·맥락 분석. 가드레일 통과 후 작업 시나리오 정의 (HS 매칭 → 관세·규제 → 바이어 추천 → 보고서 생성).
L5 Master Agent
Orchestration이 Unit Agent 호출 사슬 구성: HS Code 매칭 Agent → 외부검색 Agent (관세규제/TradeNAVI) → 바이어 추천 Agent (tradeKorea) → 생성형 Agent (보고서 초안).
L5 Orchestration
각 Agent가 RAG 조회 (Vector DB) + Tool Use (MCP)로 데이터/기능 호출. 정형 데이터는 Lakehouse, 비정형/지식은 AI Data Hub, 도메인 트랜잭션은 L3 서비스 MCP 서버.
L2 Digital Brain + L3 서비스
결과 스트리밍으로 챗 UI에 토큰 단위 출력. 동시에 대시보드 위젯에 후속 추천(예: ESG 진단, 수출실적증명 신청 가이드) 자동 추가.
L5 → L4 대시보드
전체 대화·Tool 호출·프롬프트 입출력·LLM 토큰 사용량이 관찰성 스택(Langfuse·OTel)에 트레이스로 기록되며, 가드레일 차단·민감정보 비식별 결과도 감사 로그로 적재.

7. 사업 본질 리스크 — 제안팀이 반드시 인지할 4가지

R1. 12개월 내 5계층 동시 신·증축의 일정 리스크  신규 4개 + 고도화/전환 4개 + 클라우드 랜딩존 + Multi-Agent + 대시보드를 한 계약 안에 묶음. 빅뱅 일정 시 통합 테스트 단계에서 사고 확률 급증. 단계형 이행안(공통 기반 → 핵심 Agent/대시보드 → 도메인 병행 → 통합 테스트·전환)이 평가표 "프로젝트 관리"의 사실상 통과 기준.
R2. L2 Digital Brain의 임계경로 종속  Lakehouse·AI Data Hub·메타·표준·리니지가 늦으면 L5 Agent와 L4 대시보드 전체가 슬립. M2 마일스톤(Lakehouse MVP)이 사업 성패의 첫 분기점.
R3. AI 품질·비용·종속 3중 함정  LLM 모델 종속(SFR-AP-001 명문 금지), 데이터 주권(개인정보·매출정보 외부 유출 방지), 토큰 비용 폭증을 동시에 통제해야 함. LiteLLM·vLLM·GPTCache·Langfuse 조합이 사실상 표준 답안. Hallucination 통제·평가체계(Ragas/DeepEval)가 정량 산출물로 제안서에 명시되지 않으면 평가위원이 "구체성 미달"로 감점.
R4. 도메인 깊이 부족 — 공통 약점  무역 도메인(HS Code, 관세, 수출실적증명, 지재권, ESG VSME) 전문성은 SI사 공통 약점. 무역 도메인 파트너 컨소시엄 없이 단독 제안 시 평가표 "사업수행조직 + 유사분야 수행경험"에서 격차 발생. KTNET 등 무역 정보통신 전문 파트너의 표지 노출이 차별화의 핵심.

8. 제안팀 핵심 메시지 한 페이지

  1. 이 사업은 AI 챗봇 도입이 아니다. "데이터 통합 + AI Agent + 8개 도메인 서비스 신·증축 + 클라우드 랜딩존"이 하나의 계약으로 묶인 멀티트랙 SI다.
  2. L2 Trade Digital Brain이 사업의 토대. 임계경로의 첫 분기점이고, 여기서 늦으면 L4·L5 전체가 일정 슬립.
  3. Trade Agent는 Unit Agent 27종 + Master + Orchestration의 통합 시스템. 모델 추상화·MCP·가드레일·평가체계가 표준 답안. 단순 LLM 통합으로 접근하면 평가에서 떨어진다.
  4. 평가 핵심은 25 + 20 = 45점의 "구체성". 모호 표현은 불가능 간주 룰이 박혀 있다. 정량 지표·산출물·일정·자산 인계까지 숫자로 박을 것.
  5. 경쟁력의 본질은 도메인 깊이와 운영 책임. 기술 스택은 LG CNS·메가존·kt ds 모두 비슷하게 그릴 수 있다. 무역 도메인 컨소시엄 + 12개월 사고 0 + 운영 트랜지션 보증이 차별화의 마지막 칼날.
